# Local data and DB config

Role: local development data and generated database configuration for the PHP backend. This directory must stay outside browser-public roots.

## Files

- `nichoir.sqlite`: local/dev SQLite database. It is ignored by Git and excluded from production artifacts.
- `db-config.php`: local DB config generated by `{NICHOIR_ADMIN_PATH}` > `Reglages` or by `installation/` when saving cPanel/MySQL or SQLite settings. Ignored by Git.
- `installed.lock.php`: install lock written by `installation/` after setup. Ignored by Git.
- `README.md`: this file.

## Rules

- Do not commit `*.sqlite`, `*.db`, `db-config.php`, `installed.lock.php`, dumps, backups or real customer data.
- Do not store production secrets in this directory.
- Keep only local seed/test data here.
- In production cPanel artifacts, `data/` lives under `nichoir_private/`, never `public_html/`.
- Production target is MySQL/MariaDB with private config outside `public_html`; SQLite is local/development only and is rejected when `NICHOIR_ENV=production`.
- If local DB changes are accidental, discard the SQLite file rather than documenting the changed data as source.
- Schema changes should be represented in migrations or runtime schema code, not by committing a database file.

## Local seed data

`server-php/scripts/seed_demo_dataset.php` can populate local users, credits, subscriptions, payments, tickets and notifications for local testing. Do not run that script on production data and do not publish seeded credentials in release docs.
